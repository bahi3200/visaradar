---
name: Registration Flow
description: Simple registration (name, email, password), subscription happens later from pricing page
type: feature
---
- Registration is simple: name, email, password, optional phone & telegram
- No service type or package selection during registration
- After signup, user is redirected to /pricing to subscribe
- Referral codes still work via ?ref= query param
- Subscription flow is separate via /subscribe page (choose service → package → payment)
